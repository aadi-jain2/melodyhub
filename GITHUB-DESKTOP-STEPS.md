# Quick GitHub Desktop Setup Steps

## Step 1: Add Your Repository to GitHub Desktop

1. Open **GitHub Desktop**
2. Click **"File"** â†’ **"Add Local Repository"** (or use `Ctrl+O`)
3. Click **"Choose..."** and navigate to: `C:\hubmelody-website`
4. Click **"Add Repository"**

## Step 2: Make Your First Commit

1. You should see all your files listed in the left panel
2. At the bottom left, type a commit message: `Initial commit: MelodyHub website`
3. Click **"Commit to main"** (button at bottom)

## Step 3: Publish to GitHub

1. After committing, click the **"Publish repository"** button (usually at the top)
2. A dialog will appear:
   - **Name**: `melodyhub-website` (or your preferred name)
   - **Description**: "MelodyHub - Student-led singing initiative website"
   - **Keep this code private**: Uncheck this (so you can use free GitHub Pages)
3. Click **"Publish Repository"**
4. GitHub Desktop will upload all your files to GitHub

## Step 4: Enable GitHub Pages (Free Website Hosting)

1. Go to your repository on GitHub.com (you can click "View on GitHub" in GitHub Desktop)
2. Click the **"Settings"** tab
3. In the left sidebar, click **"Pages"**
4. Under **"Source"**, select:
   - Branch: **"main"**
   - Folder: **"/ (root)"**
5. Click **"Save"**
6. Wait 1-2 minutes, then visit: `https://YOUR_USERNAME.github.io/melodyhub-website/`

## Step 5: Future Updates

Whenever you make changes to your website:

1. Make your changes in the files
2. Open GitHub Desktop
3. You'll see the changed files
4. Write a commit message (e.g., "Updated gallery images")
5. Click **"Commit to main"**
6. Click **"Push origin"** (or use `Ctrl+P`)
7. Your website will update automatically within a few minutes!

## Important Note About Videos

If GitHub Desktop shows an error about large files (videos > 100MB):
- Videos might be too large for GitHub
- You can either:
  1. Remove videos from the commit (they'll stay on your computer)
  2. Upload videos to YouTube/Vimeo and embed them instead
  3. Use GitHub LFS for large files (more advanced)
